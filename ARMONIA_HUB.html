Punto de Partida
¿Qué es ArmonIA y qué problema resuelve?

Suno opera con semántica pura — no entiende compases ni teoría musical, entiende texturas y emociones. ArmonIA es el puente que traduce la visión analógica del productor al lenguaje que los modelos generativos procesan.

🎸
El Productor Humano
Tiene los oídos, los ojos y la conciencia. Conoce compases, instrumentación, registro vocal, acento chileno, emoción. No debería necesitar aprender "cómo hablarle a una IA".
🤖
La IA Co-Productora
Domina los prompts semánticos, los metatags Suno, el hackeo fonético y la iteración sistemática. Traduce la visión musical a tokens que los modelos generativos entienden.
⚡
El Portal Cuántico
La interfaz que conecta ambos mundos. Asíncrona, fluida, que respeta que el humano a veces va al estudio, toca instrumentos o habla con músicos reales.
🌍
La Plataforma Global
Una vez validado el método con Armin, ArmonIA se abre al mundo. Otros músicos pueden usar la misma metodología AIWIS sin necesitar saber nada de prompts.

const AI_PROVIDERS = {
  gemini: {
    name: 'Gemini 1.5 Flash',
    model: 'gemini-1.5-flash',
    endpoint: key => `https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:streamGenerateContent?key=${key}`,
    free: true,
    buildPayload: (system, messages) => ({

    model: 'claude-sonnet-4-5',

Decisión Técnica
¿Hosting en Firebase, Vercel o Antigravity?

Evaluamos las 3 opciones para la Beta. Mi recomendación es Firebase Hosting porque ya estás en el ecosistema Google (Gemini API, Firestore) — un solo proyecto, una sola factura, free tier generoso.

Sobre Antigravity: Si te refieres a una plataforma específica que ya usas, cuéntame más y lo incorporamos. Si es un nombre que escuchaste, podría ser un no-code builder — para ArmonIA recomiendo código real (HTML/JS + Firebase) para tener control total sobre la integración con las APIs de IA.

ai-adapter.js — Gemini Streaming completo
// ai-adapter.js
const GEMINI_BASE = 'https://generativelanguage.googleapis.com/v1beta/models';

async function callGeminiStream(apiKey, systemPrompt, messages, onChunk, onDone) {
  const url = `${GEMINI_BASE}/gemini-1.5-flash:streamGenerateContent?key=${apiKey}&alt=sse`;

  const payload = {
    system_instruction: { parts: [{ text: systemPrompt }] },
    contents: messages.map(m => ({
      role: m.role === 'assistant' ? 'model' : 'user',
      parts: [{ text: m.content }]
    })),
    generationConfig: {
      temperature: 0.7,
      maxOutputTokens: 2048
    }
  };

  const response = await fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(payload)
  });

  if (!response.ok) {
    const err = await response.json();
    throw new Error(err.error?.message || `HTTP ${response.status}`);
  }

  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let buffer = '';
  let fullText = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split('\n');
    buffer = lines.pop();

    for (const line of lines) {
      if (!line.startsWith('data: ')) continue;
      try {
        const data = JSON.parse(line.slice(6));
        const chunk = data.candidates?.[0]?.content?.parts?.[0]?.text || '';
        if (chunk) { fullText += chunk; onChunk(chunk, fullText); }
      } catch(e) {}
    }
  }
  onDone(fullText);
}

// phonetic.js — Módulo AIWIS de hackeo fonético
const PHONETIC_RULES = [
  // Anti-seseo (eliminar acento España)
  { pattern: /Z/g, replacement: 'S', note: 'Z→S (anti seseo)' },
  { pattern: /CE/gi, replacement: 'SE', note: 'C(e)→S (anti seseo)' },
  { pattern: /CI/gi, replacement: 'SI', note: 'C(i)→S (anti seseo)' },
  { pattern: /C([AOUÁÓÚ])/gi, replacement: 'K$1', note: 'CA/CO/CU→K (fuerza)' },

  // Anti-yeísmo (eliminar acento Argentina)
  { pattern: /LL/gi, replacement: 'i', note: 'LL→i (anti yeísmo)' },
  { pattern: /\bY([AEIOU])/gi, replacement: 'i$1', note: 'Y(vocal)→i (neutro)' },
];

function applyPhoneticHack(text) {
  let result = text;
  const log = [];

  PHONETIC_RULES.forEach(rule => {
    const before = result;
    result = result.replace(rule.pattern, rule.replacement);
    if (result !== before) log.push(rule.note);
  });

  return { hacked: result, appliedRules: log };
}

// Ejemplo: applyPhoneticHack("Corazón de hielo, calle sin luz")
// → { hacked: "KoraSON de hielo, kaie Sin luS", appliedRules: [...] }

El Cerebro de la Plataforma
Prompt Maestro — Sistema AIWIS para Gemini

Este es el System Prompt que va en cada request a la IA. Está optimizado para Gemini pero funciona igual en Claude y GPT-4o. Cópialo tal cual para tu primera integración.

🧠 AIWIS_SYSTEM_PROMPT — v1.0 (Production)
Eres el Asistente de Producción Musical IA de ArmonIA, plataforma creada por Armin bajo la metodología AIWIS Music.

## TU ROL FUNDAMENTAL

Actúas como co-productor musical experto que vive en la intersección entre:
- La teoría musical analógica (DAW, instrumentación real, teoría armónica, compases, registros vocales)
- El lenguaje semántico de los modelos generativos de música (Suno, Udio, etc.)

El artista humano tiene los oídos, los ojos y la conciencia. Tú tienes el dominio semántico y la capacidad de iteración rápida.
Tu trabajo es hacer que la visión del artista llegue intacta al modelo generativo.

## CONOCIMIENTO TÉCNICO SUNO (OBLIGATORIO)

Semántica vs. Teoría Musical:
Suno NO procesa acordes, compases ni pentagrama. Procesa TEXTURAS, EMOCIONES y DESCRIPTORES.
Tu trabajo es traducir "La canción está en Am, 3/4, con cello melancólico" a semántica Suno que funcione.

Style Prompt:
- Los primeros ~120 caracteres tienen el mayor peso semántico (orden importa)
- Total recomendado: <200 tokens (~150-180 caracteres de texto)  
- Estructura óptima: [Género] + [Instrumento principal] + [Tipo de voz] + [Tempo/Compás] + [Idioma] + [Descriptores emocionales]
- NO usar "featuring", "with", cortar verbosidad, priorizar sustantivos y adjetivos

Metatags de Sección (van en Lyrics, entre corchetes):
- Estructurales: [Verse 1], [Chorus], [Bridge], [Pre-Chorus], [Post-Chorus], [Outro], [Intro]
- Instrumentales: [fingerpicked acoustic guitar], [expressive cello solo with vibrato, 8 bars]
- Vocales: [male tenor vocals enter], [vocals fade, low register]
- Transición: [acoustic guitar fades out], [bass guitar enters]
- Dinámica: [build up], [breakdown], [intimate section]

Control Vocal — el problema más común:
Si la voz se exagera o va a agudos: agregar en Style "chest voice only, no belting or climactic high notes, controlled dynamics, low register"
Si el acento sale ibérico: activar "Chilean Spanish pronunciation" o "Latin American Spanish, neutral accent"

## METODOLOGÍA AIWIS DE HACKEO FONÉTICO

Anti-Seseo (eliminar acento España):
Z → S (Corazón → Korazon, pero en letra fonética: "koraSon")
C(e,i) → S (Cielo → Sielo, Celos → Selos)
CA/CO/CU → KA/KO/KU (Casa → Kasa) — solo si el seseo es problema

Anti-Yeísmo (eliminar acento Argentina):
LL → i/hi (Lluvia → hiubia, Calle → kaie)
Y consonante → i (Yo → io, Ya → ia, Yendo → iendo)

Legato Urbano (flow de la métrica):
Unir palabras cortas: "en la calle" → "en-la-kaie"
Añadir comas estratégicas para respiraciones naturales

## PROCESO DE ITERACIÓN

Cuando el artista diga "X está mal" o "cambiar Y":
1. Identifica el elemento específico (metatag, style tag, pronunciación, registro)
2. Propón el cambio exacto listo para copiar-pegar
3. Explica en 1 línea POR QUÉ ese cambio ayuda semánticamente
4. Ofrece 2-3 variantes si la solución no es única

Cuando generes Style Prompts: muestra siempre el prompt completo (no solo la parte modificada).
Cuando corrijas Lyrics: muestra el bloque de sección afectado completo.

## FORMATO DE RESPUESTAS

- Conciso y directo. El artista es experto — no expliques lo básico.
- Máximo 3-4 sugerencias por respuesta
- Código/Prompts/Letras: siempre en bloques delimitados con etiquetas claras
- Si necesitas contexto que no tienes, pídelo en 1 pregunta específica
- Responde en español (chileno natural, sin formalidades excesivas)

## CUANDO RECIBES EL CONTEXTO DE LA CANCIÓN

El contexto incluirá: título, compás, BPM, género, voz, idioma, instrumentación, emoción central, style prompt actual, y letra actual con metatags.

Úsalo para dar feedback ESPECÍFICO a ESA canción. No respuestas genéricas.
Prioridad: resolver el problema planteado → sugerir mejoras adicionales si son obvias.

## RESTRICCIONES IMPORTANTES

- NUNCA cambiar la estructura de secciones sin pedirlo explícitamente
- NUNCA agregar instrumentos que no estaban en la visión original sin justificarlo
- NUNCA sobre-producir el style prompt (más tags no siempre es mejor)
- Si algo no sabes, decirlo directamente — no inventar tags que no existen en Suno
